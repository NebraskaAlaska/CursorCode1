# Monday Experiment Protocol — CFA alkali activation + leaching

This is the bench protocol and data-entry guide for Monday's experiments. It pairs
with the generated run sheet (`data/raw/experimental_icp/monday_experiment_plan.csv`)
and the Phase 2 release schema, so the measured data drops straight into the
pipeline.

> Scope note: this protocol covers *what to do and what to record*. It is not a
> safety document — follow your lab's SOPs and PPE requirements for NaOH handling.

## 1. Purpose

Measure how **Class C fly ash (CFA)** releases **Ca, Si, Al, Fe** (bulk matrix) and
**Sc / total REE** (trace targets) into alkaline solution as a function of NaOH
concentration, contact time, and CO₂ atmosphere. The data will be compared against
PHREEQC equilibrium predictions (Phase 2) and used later to train a correction
model (Phase 3). We are looking for the conditions that maximise **trace recovery
per unit of bulk dissolution and reagent use** (the sustainability proxies).

## 2. Variables

| Variable | Symbol / column | Levels for Monday |
|----------|-----------------|-------------------|
| NaOH molarity | `NaOH_M` | 0, 0.1, 0.25, 0.5, 1.0 M |
| Contact time | `time_min` | 10, 20, 40, 60, 90, 120 min |
| Temperature | `temperature_C` | 25 °C (held constant) |
| Liquid/solid ratio | `liquid_solid_ratio` | 5 (held constant) |
| CO₂ atmosphere | `CO2_condition` | `open` or `sealed` |
| Replicate | `replicate` | 1 (3 for the replicate-check set) |

Allowed `CO2_condition` labels (must match exactly): `open`, `sealed`, `low_CO2`,
`atm_CO2`, `unknown`.

## 3. Sample ID convention

```
CFA-NaOH{NaOH_M}M-LS{liquid_solid_ratio}-{time_min}min-{CO2_condition}-R{replicate}
```

Examples:
- `CFA-NaOH0.5M-LS5-60min-open-R1`
- `CFA-NaOH1M-LS5-60min-open-R1`
- `CFA-NaOH0M-LS5-60min-open-R1` (water blank)

Numbers are written compactly (`1.0 → 1`, `0.25 → 0.25`). Two runs that differ only
in replicate get distinct ids (`…-R1`, `…-R2`); otherwise-identical conditions share
an id and are scheduled once.

## 4. Minimum experiment matrix

Generated by `scripts/06_generate_experiment_plan.py` from four sets (duplicate
conditions are de-duplicated across sets):

1. **Time series** — `NaOH_M=0.5`, `time_min ∈ {10,20,40,60,90,120}`, open, R1.
2. **NaOH series** — `NaOH_M ∈ {0,0.1,0.25,0.5,1.0}`, `time_min=60`, open, R1.
3. **CO₂ control** — `NaOH_M=0.5`, `time_min=60`, `CO2 ∈ {sealed, open}`, R1.
4. **Replicate check** — `NaOH_M=0.5`, `time_min=60`, open, `replicate ∈ {1,2,3}`.

All share `temperature_C=25`, `liquid_solid_ratio=5`, `fly_ash_type=CFA`.

## 5. Materials / equipment checklist

- [ ] Class C fly ash, dried, weighed aliquots
- [ ] NaOH stock solutions at each target molarity (0.1, 0.25, 0.5, 1.0 M) + DI water (0 M)
- [ ] Reaction vessels — for `sealed` runs, gas-tight; for `open`, vented to lab air
- [ ] Temperature control / water bath at 25 °C
- [ ] Orbital shaker or stirrers
- [ ] Calibrated pH meter (buffers 4 / 7 / 10) and conductivity meter
- [ ] Syringe filters (record pore size in `filtration_notes`)
- [ ] Acid for sample preservation + labelled ICP vials
- [ ] **Dilution record sheet** — note every dilution factor (ICP needs it)
- [ ] PPE for NaOH (goggles, gloves, lab coat)

## 6. Step-by-step procedure

1. **Label** vessels and ICP vials with the exact `sample_id`.
2. **Weigh** the fly-ash aliquot; compute solution volume from `liquid_solid_ratio = 5`.
3. **Add activator** (NaOH at the set molarity, or DI water for 0 M). For `sealed`
   runs, close the vessel immediately; for `open`, leave vented.
4. **Record `initial_pH`** and start the timer.
5. **Agitate** at 25 °C for the set `time_min`.
6. At time, **record `final_pH` and `conductivity_mS_cm`**.
7. **Filter** the supernatant; note pore size and any clogging in `filtration_notes`.
8. **Preserve / dilute** for ICP; **write down the dilution factor**.
9. **Observe** and record any precipitate (`precipitate_observed`: `yes`/`no`) and
   appearance in `notes`.
10. Submit to ICP for Ca, Si, Al, Fe, Na, K (report as **mM**) and Sc, total REE
    (report as **ppb**).

## 7. Observations to record

- `initial_pH`, `final_pH`, `conductivity_mS_cm`
- Visible precipitate (yes/no) and colour/turbidity in `notes`
- Filtration behaviour and filter pore size in `filtration_notes`
- **Dilution factor** (in `notes`, or add a dedicated `dilution_factor` column —
  the validator warns if neither is present)
- Anything off-protocol (temperature drift, spill, re-filter) in `notes`

## 8. Entering data into the CSV

1. Copy the blank template
   `data/raw/experimental_icp/experimental_release_template.csv` to a dated file in
   the **same folder**, e.g. `data/raw/experimental_icp/2026-06-01_release.csv`.
   (You can also start from the filled run sheet — its columns already match the
   release schema, so no renaming is needed.)
2. One row per measured sample. Use the **exact `sample_id`** from the run sheet.
3. Units: `*_mM` for Ca/Si/Al/Fe/Na/K, `*_ppb` for Sc and total REE.
4. Leave any unmeasured numeric cell **blank** (becomes `NaN`) — do **not** write
   `n/a` or `0`.
5. Keep `CO2_condition` to an allowed label (§2).
6. Do **not** edit the template itself, and do not commit raw measured data unless
   instructed (it is gitignored / treated as confidential research data).

## 9. Running the scripts after data collection

```bash
# 1. (before Monday) generate the run sheet
python scripts/06_generate_experiment_plan.py

# 2. (after entering data) validate the filled CSV
python scripts/07_validate_experimental_data.py     # -> outputs/tables/experimental_validation_report.csv

# 3. compute sustainability proxies
python scripts/08_sustainability_score.py            # -> outputs/tables/sustainability_score.csv

# 4. Phase 2: compare measured vs PHREEQC (needs sample_phreeqc_map.csv for residuals)
python scripts/01_parse_phreeqc.py
python scripts/05_compare_experimental.py
```

Fix every **error** the validator reports before trusting the comparison; review
the **warnings** (especially the dilution-factor warning) and confirm they are
expected.
